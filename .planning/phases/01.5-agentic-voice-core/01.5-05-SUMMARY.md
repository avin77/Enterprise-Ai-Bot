# Phase 1.5 Plan 05: AgentLLMAdapter + Runtime Integration Summary

One-liner: AgentLLMAdapter bridges all 4 Phase 1.5 agents into a VoicePipeline-compatible adapter with instant rollback via USE_AGENTS env var.

## What Was Built

### AgentLLMAdapter (`backend/app/agents/llm_adapter.py`)

Implements the `LLMAdapter` interface with `async def generate(text, system_context)` — same signature as `RAGLLMAdapter`. Internally orchestrates:

1. `OrchestratorAgent.route()` → intent classification + routing decision
2. Branch on `routing_target`:
   - **retrieval**: `RetrievalAgent.retrieve()` → `ResponseAgent.synthesize()`
   - **tool**: `MockToolAgent.execute()` + RAG enrichment → `ResponseAgent.synthesize()`
   - **fallback**: direct `RetrievalAgent.retrieve()` → `ResponseAgent.synthesize()`
3. `emit_trace_event()` via `asyncio.create_task()` (fire-and-forget)

Confidence < 0.7 enforcement: double-checked in both `OrchestratorAgent` and `AgentLLMAdapter` (belt-and-suspenders).

### Runtime Integration (`backend/app/orchestrator/runtime.py`)

Added `_use_agents()` function and branching in `build_pipeline()`:

```
USE_AWS_MOCKS=true  → MockLLMAdapter (always wins, safe local dev)
USE_AGENTS=true     → AgentLLMAdapter (multi-agent routing)
USE_AGENTS=false    → RAGLLMAdapter (Phase 1 RAG mode — default)
```

Rollback is instantaneous: flip `USE_AGENTS=false` in ECS task definition environment variables, no code redeploy.

### E2E Tests (`tests/e2e/test_agent_llm_adapter.py`)

8 tests covering all routing paths and edge cases:

| Test | What It Validates |
|------|-------------------|
| `test_retrieval_path` | BM25 + ResponseAgent path, citation in response |
| `test_tool_path` | MockToolAgent + enrichment + synthesis |
| `test_fallback_path` | Low-confidence routes to safe retrieval |
| `test_confidence_threshold` | confidence=0.6 coerced to fallback |
| `test_trace_event_emission` | Fire-and-forget, no crashes |
| `test_empty_chunks_handling` | Safe message when no FAQ chunks found |
| `test_orchestrator_error_fallback` | Agent crash → safe retrieval fallback |
| `test_interface_compatibility` | LLMAdapter contract validated |

## Decisions Made

1. **Tool path controlled by `use_tools` constructor parameter**: MockToolAgent is only instantiated when `use_tools=True`. Default is `False` so routing decisions for "tool" target fall through to fallback until explicitly enabled.

2. **`USE_AGENTS=false` is the default**: This ensures every existing deployment (ECS, local) continues to use the stable Phase 1 RAGLLMAdapter pipeline until a deliberate opt-in.

3. **Token counts deferred**: `llm_prompt_tokens` and `llm_response_tokens` are set to 0 in Phase 1.5. Bedrock's `converse()` response includes usage metadata — Phase 3 eval gate will extract and track these.

4. **Session ID generated per call**: Phase 1.5 has no session memory (MockMemoryStore). Each `generate()` call generates a fresh UUID for trace events. Phase 2.5 will inject real session IDs.
