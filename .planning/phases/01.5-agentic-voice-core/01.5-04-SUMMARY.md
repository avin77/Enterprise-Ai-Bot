# Phase 1.5 Plan 04: Agent Trace Event Emission — Summary

**One-liner:** Metadata-only CloudWatch trace events with intent confusion matrix and pattern-based grounding detection, wired fire-and-forget into VoicePipeline.

## What Was Built

### backend/app/agents/tracer.py (new, 298 lines)

**TraceEvent dataclass** — full trace schema with all required fields:
- `session_id`, `turn_id` (UUID), `intent`, `intent_confidence`
- `routing_target` (retrieval|tool|fallback)
- `retrieved_doc_ids` — source doc IDs from retrieval stage
- `llm_prompt_tokens`, `llm_response_tokens`, `llm_latency_ms`
- `tool_calls` — tool names and results (no sensitive data)
- `total_latency_ms`, `fallback_triggered`
- `grounded`, `grounding_signal` — source attribution signals
- `timestamp` — ISO 8601 with Z

**emit_trace_event()** — async function that:
- Builds TraceEvent with current UTC timestamp
- Serializes to JSON via `asdict()`
- Logs via `logger.info(json.dumps(event))` to `voicebot.traces` logger
- ECS awslogs driver picks up all Python logging automatically
- Swallows all errors — logging failure must not crash voice turns

**IntentConfusionMatrix** — class for eval-time analytics:
- `add_prediction(turn_id, predicted, gold_label)` — accumulates predictions
- `compute_metrics()` — returns per-intent precision, recall, F1, support counts
- `publish_to_cloudwatch(region, namespace)` — publishes to `voicebot/agents` namespace
- Handles max-20-items-per-call CloudWatch batch limit
- Never raises on publish failure

### backend/app/orchestrator/pipeline.py (extended)

- `_detect_grounding(response_text)` — regex pattern detection:
  - `"According to"` → `(True, "has_source_attribution")`
  - `"I don't have"` → `(False, "no_sources")`
  - default → `(True, "ambiguous")`
- `_fire_trace()` — wraps `asyncio.create_task(_emit_trace_event(...))` fire-and-forget
- `session_id` parameter added to `run_roundtrip()` and `run_text_turn()`
- Turn UUID generated via `uuid4()` per turn
- `RuntimeError` (no event loop) caught silently for sync test contexts

### backend/app/agents/__init__.py (updated)

- `emit_trace_event` and `IntentConfusionMatrix` added to `__all__`
- Import path: `from backend.app.agents import emit_trace_event, IntentConfusionMatrix`

## CloudWatch Logs Insights Queries (Documented)

```
# Intent distribution
fields @timestamp, intent, intent_confidence, routing_target, fallback_triggered
| filter event_type = "agent_trace"
| stats count() by intent
| sort count desc

# Fallback analysis
fields @timestamp, intent, fallback_triggered, total_latency_ms
| filter event_type = "agent_trace" and fallback_triggered = true
| stats count() as fallback_count, avg(total_latency_ms) as avg_latency

# Grounding signal breakdown
fields @timestamp, grounding_signal, grounded
| filter event_type = "agent_trace"
| stats count() by grounding_signal
```
