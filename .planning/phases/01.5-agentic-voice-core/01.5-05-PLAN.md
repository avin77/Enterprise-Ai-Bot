  <files>backend/app/agents/llm_adapter.py</files>
  <action>
    Create AgentLLMAdapter class that implements LLMAdapter interface and orchestrates the 3-agent pipeline:

    1. Interface requirement:
       - Must implement `async def generate(text: str, system_context: dict) -> str` (same signature as RAGLLMAdapter)
       - This is the contract that VoicePipeline expects from any LLM adapter

    2. Constructor:
       ```python
       def __init__(
           self,
           knowledge_adapter: KnowledgeAdapter,
           aws_clients: AwsClientBundle,
           use_memory: bool = False,
           use_tools: bool = False,
       ):
       ```
       - Store all adapters and clients as instance variables
       - Create agent instances:
         * self._orchestrator = OrchestratorAgent(aws_clients)
         * self._retrieval = RetrievalAgent(knowledge_adapter)
         * self._response = ResponseAgent(aws_clients)
       - Create optional agents:
         * If use_memory: self._memory = MockMemoryStore()
         * If use_tools: self._tool = MockToolAgent()
       - Initialize tracer module

    3. Main orchestration method:
       ```python
       async def generate(self, text: str, system_context: dict) -> str:
       ```
       - Extract session_id from system_context (or generate new UUID)
       - Call self._orchestrator.route(text, history=[]) → RoutingDecision
       - Enforce fallback: if routing_decision.confidence < 0.7, set routing_target='fallback'
       - Branch on routing_target:

         **retrieval path:**
         - Call self._retrieval.retrieve(text, routing_decision.intent, top_k=3)
         - Get chunks: list[ChunkResult]
         - Call self._response.synthesize(text, chunks)
         - Return response

         **tool path:**
         - Call self._tool.execute(routing_decision.intent, {}) → ToolResult
         - Get tool_data and top-3 RAG chunks for enrichment
         - Call self._response.synthesize(text, chunks_enriched_with_tool_data)
         - Return response

         **fallback path:**
         - Skip Orchestrator routing, go directly to retrieval
         - Call self._retrieval.retrieve(text, intent="", top_k=3)
         - Call self._response.synthesize(text, chunks)
         - Return response

    4. Trace event emission (non-blocking):
       - Before returning response, collect trace data:
         * session_id, turn_id (UUID), intent, intent_confidence from routing_decision
         * routing_target from routing_decision
         * retrieved_doc_ids from chunks
         * llm_prompt_tokens, llm_response_tokens, llm_latency_ms (from agent calls — track during execution)
         * tool_calls (empty for Phase 1.5 MockToolAgent)
         * total_latency_ms (time from start to finish)
         * fallback_triggered (if confidence < 0.7)
         * grounded (check if response has citations)
       - Call: asyncio.create_task(emit_trace_event(...)) (fire-and-forget)

    5. Error handling:
       - If any agent call fails: log error, fall back to safe retrieval
       - Never crash — always attempt to return a response (even "I'm having trouble...")
       - Trace event emission failures: log only, don't crash

    6. Imports:
       - from backend.app.agents.orchestrator import OrchestratorAgent
       - from backend.app.agents.retrieval import RetrievalAgent
       - from backend.app.agents.response import ResponseAgent
       - from backend.app.agents.memory import MockMemoryStore
       - from backend.app.agents.tool import MockToolAgent
       - from backend.app.agents.tracer import emit_trace_event
       - from backend.app.services.llm import LLMAdapter
       - from backend.app.services.knowledge import KnowledgeAdapter
       - from backend.app.services.aws_clients import AwsClientBundle
       - asyncio, logging, json, uuid, time
