## Test Breakdown by Dimension

### Dimension 1: Happy Path (Retrieval Routing)
- `test_retrieval_path` — Generate query → Orchestrator routes to 'retrieval' → chunks retrieved → ResponseAgent synthesizes with citations

### Dimension 2: Guardrails Integration
- `test_guardrails_in_response_agent` — Verify Phase 2 guardrails (build_response_system_prompt) are used in ResponseAgent system prompt

### Dimension 3: Observability (Trace Events)
- `test_trace_event_emitted` — Verify emit_trace_event() is called after generate(), trace contains session_id, turn_id, intent, confidence, routing_target, tokens, latency

### Dimension 4: Resilience (Fallback & Errors)
- `test_low_confidence_triggers_fallback` — Confidence < 0.7 forces routing_target='fallback', skips Orchestrator routing
- `test_orchestrator_error_graceful_fallback` — OrchestratorAgent exception → falls back to safe retrieval, no crash

### Dimension 5: Phase Integration (Tool Path)
- `test_tool_path_with_rag_enrichment` — Tool routing → MockToolAgent canned data + top-3 RAG chunks → ResponseAgent synthesis

### Dimension 6: Edge Cases
- `test_empty_chunks_no_crash` — Knowledge adapter returns empty list → ResponseAgent still responds ("I don't have that info...")
- `test_invalid_routing_decision_json` — OrchestratorAgent returns malformed JSON → fallback to safe retrieval

### Dimension 7: Performance
- `test_response_latency_under_2s` — AgentLLMAdapter.generate() completes in < 2000ms (p95 target from Phase 1 eval)

### Dimension 8: Nyquist Compliance
- VALIDATION.md itself (this file) — Defines test strategy, Wave 0 requirements, per-task verification, manual-only behaviors
- Test infrastructure: pytest 7.x with async support, conftest.py fixtures, e2e test suite
